Installation Guide

Get MRF Compiler up and running on your system

Linux Installation

Prerequisites

bash 
# Install build tools
sudo apt-get update
sudo apt-get install -y build-essential g++ make git

# For Ubuntu/Debian
sudo apt-get install -y libc6-dev

# For Fedora/RHEL
sudo dnf install -y gcc-c++ make git

Build from Source

bash 
# Clone the repository
git clone https://github.com/Sapana-Micro-Software/MRFcompiler.git
cd MRFcompiler

# Build the compiler
make

# Optional: Install to system path
sudo make install

Verify Installation

bash 
./mrf_compiler --help

macOS Installation

Option 1: Using Homebrew (Recommended)

Install Homebrew (if not already installed)

bash 
# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Install Compiler via Homebrew

Choose one of the following options:

Option A: LLVM (clang++)
bash 
# Install LLVM via Homebrew
brew install llvm

# Verify installation
/opt/homebrew/opt/llvm/bin/clang++ --version
Option B: GCC
bash 
# Install GCC via Homebrew
brew install gcc

# Verify installation (version may vary: g++-13, g++-12, etc.)
brew list gcc | grep bin/g++

Option 2: Using Xcode Command Line Tools

bash 
# Install Xcode Command Line Tools (includes Apple's clang)
xcode-select --install

# Verify installation
clang++ --version

Build from Source

bash 
# Clone the repository
git clone https://github.com/Sapana-Micro-Software/MRFcompiler.git
cd MRFcompiler

# Check which compiler will be used
make check-compiler

# Build the compiler (automatically detects Homebrew LLVM/GCC or system clang)
make

# Optional: Install to system path
sudo make install

# Or install to Homebrew prefix (no sudo needed)
make install INSTALL_PREFIX=/opt/homebrew

Verify Installation

bash 
# Test the compiler
./mrf_compiler --help

# Run a test
make test

Troubleshooting

bash 
# If Homebrew LLVM is not detected, check installation path
ls -la /opt/homebrew/opt/llvm/bin/clang++
ls -la /usr/local/opt/llvm/bin/clang++

# If Homebrew GCC is not detected, check available versions
ls -la /opt/homebrew/bin/g++-*
ls -la /usr/local/bin/g++-*

# Force use of specific compiler
make CXX=/opt/homebrew/opt/llvm/bin/clang++
# or
make CXX=/opt/homebrew/bin/g++-13

Windows Installation

Using WSL (Recommended)

We recommend using Windows Subsystem for Linux (WSL) for the best experience.

bash 
# Install WSL and Ubuntu
wsl --install

# Then follow Linux installation instructions

Using MinGW/MSYS2

bash 
# Install MSYS2 from https://www.msys2.org/
# Then install build tools
pacman -S mingw-w64-x86_64-gcc make git

# Clone and build
git clone https://github.com/Sapana-Micro-Software/MRFcompiler.git
cd MRFcompiler
make

Troubleshooting

Build Errors

If you encounter build errors, ensure you have:

  • C++11 compatible compiler (GCC 4.8+, Clang 3.3+)
  • Make utility installed
  • Sufficient permissions for installation

Runtime Errors

If the compiler fails to run:

  • Check that the executable has proper permissions: chmod +x mrf_compiler
  • Verify input file format matches the expected syntax
  • Check that output directory is writable

Framework Import Errors

If generated Python code fails to import:

  • Ensure the framework package is installed in your Python environment
  • Check Python version compatibility (Python 3.7+ recommended)
  • Verify virtual environment is activated if using one
